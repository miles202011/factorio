# 按钮竖向文字方案记录

## 目标

在 Factorio 斗地主 UI 里，让扑克牌显示为竖向牌面：

```text
A
♠
```

大小王显示为：

```text
大
王
```

同时希望保留：

- 牌面颜色接近 Factorio 原生按钮灰色。
- 点击整张牌可以选牌。
- 牌面不要出现物品槽方格纹理。
- HTML 预览页和游戏内 UI 尽量一致。

## 已尝试方案

### 1. 单个 button + caption 换行

做法：

```lua
button.caption = "A\n♠"
```

问题：

- Factorio 的 button caption 不可靠支持换行。
- 实际显示会变回横向，例如 `A♠`。

结论：

- 颜色和点击体验最好。
- 但不能稳定实现真正竖向文字。

### 2. frame + 多个 label

做法：

- 一张牌用一个 `frame`。
- 里面放多个 `label`。
- 点数一行，花色一行。

优点：

- 真正竖向。
- 花色位置可控。
- 大小王也能自然竖向显示。

问题：

- 普通场景脚本不能自定义新的 GUI style。
- 只能用 Factorio 现成 frame 样式。
- `slot_button_deep_frame` 有物品槽方格纹理。
- `subheader_frame` 偏深，不像底牌按钮。
- `inside_shallow_frame` 颜色也和按钮牌不一致。

结论：

- 排版最好。
- 颜色很难和原生按钮完全一致。

### 3. 一张牌拆成上下两个 button

做法：

- 一张牌外面放一个竖向 flow。
- 上半部分是点数按钮。
- 下半部分是花色按钮。

例如：

```text
[ A ]
[ ♠ ]
```

优点：

- 每一段都是原生 button，颜色接近底牌按钮。
- 可以实现竖向。
- 点击事件可以通过父级名字兜底处理。

问题：

- 视觉上像一张牌被切成两块。
- 不够像完整扑克牌。

结论：

- 可用，但不好看。

### 4. 用 `A︱♠` 这类字符模拟

问题：

- 这本质仍然是横向。
- 只是用竖线分隔，不是真正竖排。

结论：

- 不推荐。

## 当前判断

在当前项目作为 Factorio scenario 的前提下，不能稳定注册自定义 GUI style，所以没有办法同时完美满足：

- 原生按钮灰色。
- 一个完整牌面。
- 真竖向文字。
- 整张牌点击。

如果继续保持 scenario 结构，比较现实的选择是：

1. 回到单按钮横排：最稳定、最好看、最像原生按钮。
2. 用 frame + label：真竖向，但需要接受牌面颜色和按钮不完全一致。
3. 统一所有牌面，包括底牌背面，也用同一套 frame + label：不追求像原生按钮，但整桌风格统一。

## 最推荐的后续方案

### 如果继续做 scenario

推荐：

- 用 `frame + label` 做真竖向牌面。
- 底牌背面也改成同样的牌面组件。
- 不再强行追求和原生按钮完全一致，而是追求整桌牌面统一。

理由：

- 竖向排版可控。
- 牌面不会被切开。
- 大小王显示自然。
- 所有牌面统一后，颜色差异问题会减轻。

### 如果愿意改成完整 mod

推荐：

- 在 `data.lua` 注册自定义 `ddz_card_frame` 样式。
- 继承或仿制原生按钮灰色图形。
- runtime 里用 `frame + label`。

这是理论上最理想的方案：

- 颜色可控。
- 文字竖向。
- 牌面完整。
- 点击逻辑也可控。

缺点：

- 工程结构更复杂。
- 不再只是普通 scenario 脚本。

### 如果追求最佳视觉

推荐：

- 做牌面图片或 sprite。
- 每张牌是一张小图。
- 按钮只负责点击，视觉由图片控制。

优点：

- 最好看。
- 字体、花色、位置完全可控。

缺点：

- 需要准备牌面资源。
- 资源管理和加载更复杂。
